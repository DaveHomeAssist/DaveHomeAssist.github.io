3A. Sort Controls
    - Add compact sort dropdown beside view toggle
    - Options: Name A-Z, Featured First, Status, Hosting, Category

    - Wire into getFiltered() → getFilteredAndSorted()

    - Save view, category, status, hub filter, sort to localStorage

    Owner: Dave

    - Sync search/filter/view/sort to query params
      e.g. ?view=list&category=dev-tools&search=prompt&sort=name
    - Read params on init to restore state (overrides localStorage)
    - Update URL on every state change via replaceState (no history spam)

    Milestone: Filter states are shareable via URL
    Owner: Dave

3D. Interactive Dependency Map
    - On hover/click of a dependency card, highlight related project cards
    - Add anchor links from dependency items to their project cards
    - Optional: click dependency to filter to that relationship

    Milestone: Dependency map is connected to the project grid
    Owner: Dave

    Est. timeline: 4-6 hours across 2-3 sessions

PHASE 4 — RENDERING & SECURITY HARDENING               [COMPLETED]
================================================================================
Goal: Replace innerHTML-driven rendering with safer patterns

4A. DOM Rendering Migration
    - Replace renderCard() string templates with createElement/textContent
    - Replace renderRow() string templates with createElement/textContent
    - Replace renderDeps() string templates with createElement/textContent
    - Use DocumentFragment for batch DOM insertion
    - Remove all inline onclick handlers; use delegated event listeners
      on .project-grid and .project-list containers

    Milestone: Zero innerHTML assignments with user/manifest data
    Owner: Dave

4B. Manifest Validation
    - Add lightweight schema validation at load time
    - Check required fields: meta.owner, projects[].id, projects[].name
    - On validation failure: show user-facing error state, fall back to embedded
    - Log validation warnings for optional missing fields

    Milestone: Malformed manifest shows clear error instead of silent breakage
    Owner: Dave

4C. Search Debounce
    - Debounce search input by 150-200ms
    - Prevents full DOM rebuild on every keystroke
    - Simple setTimeout/clearTimeout pattern (no library needed)

    Milestone: Search input doesn't trigger render on every character
    Owner: Dave

    Est. timeline: 4-6 hours across 2-3 sessions
